📚 学习任务指导书

项目六：监控、测试并优化 MySQL 性能

⏱️ 6 课时 🏢 天天电器商场 📂 数据库运维与管理
🏢 业务场景
⚡ 天天电器商场 — 夏季销售旺季性能危机
背景 夏天空调销售旺季，OA系统页面卡顿，假死机频发，严重影响业务运转。
反馈 业务系统管理员反馈：按电话号码尾号模糊查询时出现假死状态，大量查询请求堆积。
要求 技术主管要求：监控数据库性能，找出瓶颈，优化SQL语句，保障系统稳定运行。
🎯 教学目标
📖 知识目标
  • 掌握性能监控常用指标（QPS/TPS/缓冲池命中率）
  • 掌握 sysbench / mysqlslap 压力测试方法
  • 掌握 EXPLAIN 执行计划分析方法
  • 能使用 MONyog 进行数据库监控
🛠️ 能力目标
  • 能完成 MySQL 压力测试并出具测试报告
  • 能使用 EXPLAIN 分析 SQL 执行计划
  • 能针对性能瓶颈进行 SQL 语句优化
🌱 素养目标
  • 培养数据驱动的运维思维
  • 养成系统化的性能分析习惯
  • 建立监控预警意识
🏛️ 思政目标
  • 树立质量意识与工匠精神
  • 理解技术服务于业务的价值
📋 任务总览
任务 6-1 ⭐ 核心
安装 MONyog 工具监控 MySQL 服务器
⏱️ 约 30 分钟
下载安装 MONyog 监控工具，连接 MySQL 服务器，配置监控面板，查看 QPS、TPS、连接数、缓冲池命中率等关键性能指标。
任务 6-2 ⭐ 核心
对 MySQL 数据库读写性能调优
⏱️ 约 30 分钟
通过调整 innodb_buffer_pool_size、tmp_table_size、max_connections 等配置参数，优化 MySQL 读写性能，并通过 MONyog 验证效果。
任务 6-3 ⭐ 核心
创建 MySQL 数据库压力测试报告
⏱️ 约 40 分钟
使用 sysbench 或 mysqlslap 对 MySQL 进行压力测试，记录 QPS、TPS、响应时间等数据，撰写完整的性能测试报告。
任务 6-4 ⭐ 核心
使用 EXPLAIN 工具分析并优化单表 SQL 查询
⏱️ 约 30 分钟
针对天天电器商场 OA 系统中的单表查询，使用 EXPLAIN 分析执行计划，重点关注 type、key、rows、Extra 字段，添加合适索引进行优化。
任务 6-5 ⭐ 核心
使用 EXPLAIN 工具分析并优化多表 SQL 查询
⏱️ 约 40 分钟
分析涉及 JOIN 的多表查询语句，运用联合索引最左前缀原则和覆盖索引技术进行优化，对比优化前后执行计划。
任务 6-6 🌱 拓展
申请及使用阿里云 RDS 数据库
⏱️ 约 45 分钟
了解阿里云 RDS 云数据库的申请流程，完成实例创建与连接配置，体验云端数据库的监控与运维管理功能。
任务 6-7 📝 实训
MySQL 及阿里云 RDS 数据库运维任务
⏱️ 约 30 分钟
综合运用前面所学知识，完成 MySQL 本地数据库和阿里云 RDS 数据库的综合运维操作，包括备份、监控、性能优化等。
💡 知识要点速查
📊 性能监控指标
QPS 每秒查询数，衡量数据库查询吞吐量
TPS 每秒事务数，衡量数据库事务处理能力
Connections 当前连接数与最大连接数
Buffer Pool InnoDB 缓冲池命中率，理想值 > 99%
🔧 监控工具
mysqladmin MySQL 内置管理工具，可查看状态变量
MONyog 企业级 MySQL 监控工具，可视化仪表盘
SHOW STATUS 查看 MySQL 运行时状态变量
SHOW PROCESSLIST 查看当前连接的线程列表
⚡ 压测工具
mysqlslap MySQL 内置压测工具，模拟并发查询
sysbench 开源跨平台基准测试工具
oltp_read_only sysbench 只读测试模式
oltp_read_write sysbench 读写混合测试模式
🔍 EXPLAIN 关键字段
type 访问类型，从优到差：system > const > eq_ref > ref > range > index > ALL
key 实际使用的索引名称
rows 预估扫描的行数，越少越好
Extra 额外信息，关注 Using index / Using filesort / Using temporary
🏷️ 索引优化
最左前缀 联合索引遵循最左前缀匹配原则
覆盖索引 查询字段全部被索引包含，避免回表
索引下推 MySQL 5.6+ 将 WHERE 条件下推到存储引擎层
避免失效 避免对索引列使用函数、隐式类型转换、LIKE '%xx'
⚙️ 关键配置参数
innodb_buffer_pool_size InnoDB 缓冲池大小，建议为物理内存 60%-70%
tmp_table_size 临时表大小上限
max_connections 最大连接数
query_cache_size 查询缓存大小（MySQL 8.0 已移除）
