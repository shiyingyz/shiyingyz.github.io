P2 · 基础

安装和配置MySQL数据库

📅 4课时
⭐ 任务2-1 在Linux系统下安装配置MySQL 核心

【任务环境】

• 虚拟机：CentOS 7.x

• MySQL版本：MySQL 8.0

【任务步骤】

步骤1：检查并卸载mariadb
rpm -qa | grep mariadb rpm -e --nodeps mariadb-libs-xxx
步骤2：下载并安装MySQL Yum Repository
wget https://dev.mysql.com/get/mysql80-community-release-el7-7.noarch.rpm rpm -ivh mysql80-community-release-el7-7.noarch.rpm
步骤3：安装MySQL服务器
yum install -y mysql-community-server
步骤4：启动MySQL服务
systemctl start mysqld systemctl enable mysqld
步骤5：获取临时密码并登录
grep 'temporary password' /var/log/mysqld.log mysql -u root -p
步骤6：修改root密码
ALTER USER 'root'@'localhost' IDENTIFIED BY 'YourNewPass123!';
📝 任务2-2 登录、退出MySQL服务器

【任务步骤】

步骤1：本机登录MySQL
mysql -u root -p
步骤2：查看MySQL版本
SELECT VERSION(); SELECT NOW(); SELECT USER();
步骤3：退出MySQL
EXIT; QUIT; \q
📝 任务2-3 在Windows系统下安装配置MySQL

【任务步骤】

步骤1：下载MySQL安装包

访问 https://dev.mysql.com/downloads/mysql/ 下载Windows版本

选择 Windows (x86, 64-bit), MSI Installer

步骤2：运行安装向导

双击.msi文件，选择"Custom"安装类型

选择安装路径，建议使用 D:\MySQL

步骤3：配置MySQL

选择"Standalone MySQL Server"

设置root密码

设置服务名称和端口（默认3306）

步骤4：验证安装
mysql -u root -p SHOW DATABASES;
📝 任务2-4 第三方运维平台下的自动化部署

【任务环境】

• 云平台或Docker环境

【任务步骤】

步骤1：使用Docker安装MySQL
docker pull mysql:8.0
步骤2：运行MySQL容器
docker run -d --name mysql8 -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 mysql:8.0
步骤3：连接测试
docker exec -it mysql8 mysql -u root -p

P3 · 安全

管理MySQL权限与安全

📅 4课时
⭐ 任务3-1 建立MySQL普通账户并登录 核心

【任务步骤】

步骤1：创建新用户
CREATE USER 'testuser'@'localhost' IDENTIFIED BY 'Test123456!'; CREATE USER 'testuser'@'%' IDENTIFIED BY 'Test123456!';
步骤2：查看用户信息
SELECT User, Host FROM mysql.user;
步骤3：使用新账户登录
EXIT; mysql -u testuser -p
步骤4：删除测试用户
DROP USER 'testuser'@'localhost'; DROP USER 'testuser'@'%';
⭐ 任务3-2 授权数据库的远程访问 核心

【任务步骤】

步骤1：创建远程登录用户
CREATE USER 'remote_user'@'%' IDENTIFIED BY 'Remote123!';
步骤2：授予所有权限
GRANT ALL PRIVILEGES ON *.* TO 'remote_user'@'%'; FLUSH PRIVILEGES;
步骤3：配置防火墙
firewall-cmd --add-port=3306/tcp --permanent firewall-cmd --reload
步骤4：修改绑定地址
vim /etc/my.cnf bind-address=0.0.0.0
步骤5：远程连接测试
mysql -h 服务器IP -u remote_user -p
📝 任务3-3 授权数据库表的读写操作

【任务步骤】

步骤1：创建业务数据库
CREATE DATABASE shop_db;
步骤2：创建应用用户
CREATE USER 'app_user'@'localhost' IDENTIFIED BY 'App123456!';
步骤3：授予特定数据库权限
GRANT SELECT, INSERT, UPDATE, DELETE ON shop_db.* TO 'app_user'@'localhost'; FLUSH PRIVILEGES;
步骤4：验证权限
SHOW GRANTS FOR 'app_user'@'localhost';
📝 任务3-4 收回MySQL普通账户权限

【任务步骤】

步骤1：查看用户权限
SHOW GRANTS FOR 'app_user'@'localhost';
步骤2：收回特定权限
REVOKE DELETE ON shop_db.* FROM 'app_user'@'localhost';
步骤3：收回所有权限
REVOKE ALL PRIVILEGES ON shop_db.* FROM 'app_user'@'localhost'; FLUSH PRIVILEGES;

📅 6课时
⭐ 任务4-1 通过错误日志查看MySQL故障原因 核心

【任务步骤】

步骤1：定位错误日志位置

步骤3：模拟故障并分析

⭐ 任务4-2 记录客户端连接错误信息 核心

【任务步骤】

步骤1：启用连接日志

步骤2：查看连接日志
tail -f /var/log/mysql/general.log
⭐ 任务4-3 使用mysqldumpslow统计分析慢查询 核心

【任务步骤】

步骤1：开启慢查询日志

步骤2：执行慢查询测试
SELECT SLEEP(3); SELECT * FROM large_table;
步骤3：使用mysqldumpslow分析
mysqldumpslow -s t /var/log/mysql/slow.log mysqldumpslow -s c -t 10 /var/log/mysql/slow.log

步骤3：基于时间点恢复
mysqlbinlog --stop-datetime="2024-01-01 12:00:00" mysql-bin.000001 | mysql -u root -p

P5 · 备份

备份与恢复MySQL数据库

📅 6课时
⭐ 任务5-1 使用mysqldump备份和恢复数据 核心

【任务步骤】

步骤1：创建测试数据库
CREATE DATABASE backup_test; USE backup_test; CREATE TABLE users (id INT PRIMARY KEY, name VARCHAR(50)); INSERT INTO users VALUES (1, '张三'), (2, '李四');
步骤2：备份单个数据库
mysqldump -u root -p backup_test > /tmp/backup_test.sql
步骤3：备份所有数据库
mysqldump -u root -p --all-databases > /tmp/all_databases.sql
步骤4：恢复数据库
mysql -u root -p backup_test < /tmp/backup_test.sql
📝 任务5-2 使用XtraBackup备份和恢复

【任务步骤】

步骤1：安装XtraBackup
yum install percona-xtrabackup-24 -y
步骤2：全量备份
innobackupex --user=root --password=123456 /backup/
步骤3：准备备份
innobackupex --apply-log /backup/2024-01-01_xx-xx-xx/
步骤4：恢复数据
innobackupex --copy-back /backup/2024-01-01_xx-xx-xx/
📝 任务5-3 使用mysqldump+crontab自动备份

【任务步骤】

步骤1：创建备份脚本
vim /root/mysql_backup.sh #!/bin/bash date=`date +%Y%m%d%H%M%S` mysqldump -u root -p'123456' --all-databases > /backup/mysql_$date.sql find /backup -name "*.sql" -mtime +7 -delete
步骤2：设置执行权限
chmod +x /root/mysql_backup.sh
步骤3：配置定时任务
crontab -e 0 2 * * * /root/mysql_backup.sh
📝 任务5-4 迁移MySQL数据库

【任务步骤】

步骤1：备份源数据库
mysqldump -u root -p shop_db > shop_db.sql
步骤2：传输到目标服务器
scp shop_db.sql root@目标IP:/tmp/
步骤3：在目标服务器恢复
mysql -u root -p -e "CREATE DATABASE shop_db;" mysql -u root -p shop_db < /tmp/shop_db.sql
📝 任务5-5 迁移MySQL表数据

【任务步骤】

步骤1：导出指定表
mysqldump -u root -p shop_db orders > orders.sql
步骤2：导入到目标表
mysql -u root -p shop_db < orders.sql

P7 · 复制

MySQL主从复制

📅 6课时
⭐ 任务7-1 在Linux系统下建立MySQL主从复制 核心

【环境说明】

• 主库：CentOS 7 + MySQL 8.0 (192.168.1.100)

• 从库：CentOS 7 + MySQL 8.0 (192.168.1.101)

【主库配置步骤】

步骤1：配置主库my.cnf
vim /etc/my.cnf [mysqld] server-id=1 log-bin=mysql-bin
步骤2：创建复制账户
CREATE USER 'repl'@'%' IDENTIFIED BY 'Repl123!'; GRANT REPLICATION SLAVE ON *.* TO 'repl'@'%'; FLUSH PRIVILEGES;
步骤3：获取binlog位置
SHOW MASTER STATUS;

【从库配置步骤】

步骤4：配置从库my.cnf
vim /etc/my.cnf [mysqld] server-id=2 relay-log=mysql-relay-bin
步骤5：配置复制连接
CHANGE MASTER TO MASTER_HOST='192.168.1.100', MASTER_USER='repl', MASTER_PASSWORD='Repl123!', MASTER_LOG_FILE='mysql-bin.000001', MASTER_LOG_POS=154;
步骤6：启动复制并验证
START SLAVE; SHOW SLAVE STATUS\G
⭐ 任务7-2 配置MySQL半同步复制 核心

【任务步骤】

步骤1：主库安装半同步插件
INSTALL PLUGIN rpl_semi_sync_master SONAME 'semisync_master.so';
步骤2：启用半同步
SET GLOBAL rpl_semi_sync_master_enabled = ON;
步骤3：从库安装插件
INSTALL PLUGIN rpl_semi_sync_slave SONAME 'semisync_slave.so'; SET GLOBAL rpl_semi_sync_slave_enabled = ON;
📝 任务7-3 配置MySQL并行复制

【任务步骤】

步骤1：设置并行复制
SET GLOBAL slave_parallel_workers = 4; SET GLOBAL slave_parallel_type = 'LOGICAL_CLOCK';
步骤2：重启从库
STOP SLAVE; START SLAVE;
📝 任务7-4 基于GTID建立MySQL主从复制

【任务步骤】

步骤1：主库启用GTID
vim /etc/my.cnf gtid_mode=ON enforce_gtid_consistency=ON
步骤2：从库启用GTID
gtid_mode=ON enforce_gtid_consistency=ON log-slave-updates=ON
步骤3：配置GTID复制
CHANGE MASTER TO MASTER_AUTO_POSITION=1 START SLAVE;
