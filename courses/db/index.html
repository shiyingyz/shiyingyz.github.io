📖 课程简介

本课程是计算机软件技术专业（大数据应用开发方向）专业群核心课程。以 MySQL 8.0 为主线，结合 Redis 缓存架构，覆盖从安装部署到智能运维的完整生命周期。课程采用"教学做一体化"，所有理论均配套实训任务。

🎯 总体目标

掌握 MySQL 及 Redis 数据库的基本运维、设计架构、问题定位与智能诊断技能，熟练使用 goInception 等自动化运维平台，使学员能够胜任高级数据库 DBA 工作，为企业"天天电器商场"业务系统保驾护航。

  • 计算机网络基础

项目一 了解数据库运维工作 ⏱ 4课时
💼 业务场景 · 天天电器商场
天天电器商场是一家连锁零售企业，业务系统每天处理数万笔订单。随着业务量增长，数据库频繁出现性能瓶颈，DBA 老张每天忙得团团转。如果你是即将入职的数据库工程师，你需要提前了解这个岗位到底做什么，以及需要哪些核心能力？

  • 主流产品：MySQL/Oracle/MongoDB
  • DBA 职责：安装/监控/备份/优化/安全
  • 运维演进：人工→工具→产品→智能

项目三 管理 MySQL 权限与安全 ⏱ 4课时
💼 业务场景 · 天天电器商场
天天电器商场有多个业务部门需要访问数据库：开发部需要读写权限，财务部只需要查询权限，市场部需要报表数据但不能修改订单。数据库工程师需要为每个部门创建独立账户，精确控制权限，确保数据安全。

🎯 教学目标

知识目标能力目标
  • 掌握权限表结构（mysql.user / db / tables_priv）
  • 掌握 CREATE USER / GRANT / REVOKE
  • 理解数据/结构/管理权限区别
  • 能创建普通用户账户
  • 能配置远程访问
  • 能精细授权与收回

⚡ 关键命令速查

CREATE USER '用户名'@'主机' IDENTIFIED BY '密码';
GRANT SELECT ON db.表 TO '用户'@'主机';
REVOKE INSERT ON db.表 FROM '用户'@'主机';
SHOW GRANTS FOR '用户'@'主机';
FLUSH PRIVILEGES;
1

项目四 分析 MySQL 日志 ⏱ 6课时
💼 业务场景 · 天天电器商场
天天电器商场的 OA 系统突然变慢，数据库查询响应时间从 0.5 秒飙升到 8 秒。业务部门纷纷投诉。技术主管要求你：① 找出哪个查询最慢；② 如果有数据被误删，能否恢复？这两件事都和 MySQL 日志有关！

🎯 教学目标

知识目标能力目标
  • 掌握错误日志/查询日志/慢查询日志/二进制日志
  • 能用 mysqldumpslow 分析慢查询
  • 能用 mysqlbinlog 时间点恢复
  • 能通过错误日志排查启动失败

⚡ 关键命令速查

tail -30 /var/log/mysqld.log
SET GLOBAL slow_query_log=ON;
mysqldumpslow -t 5 /var/log/mysql-slow.log
SHOW MASTER STATUS;
mysqlbinlog --stop-datetime="时间" bin | mysql

任务 4-3：使用 mysqldumpslow 统计分析慢查询

⭐ 核心任务 约 25 分钟
任务说明：开启慢查询日志，用 SLEEP() 模拟慢 SQL，用 mysqldumpslow 分析出耗时最多的查询。
跟着做
1

2
SELECT SLEEP(3), '慢查询测试';
3
mysqldumpslow -t 5 /var/log/mysql-slow.log
4
记录结果：Count、Time、Query 填写到实验报告
3

任务 4-4：使用 mysqlbinlog 通过时间点恢复数据

⭐ 核心任务 约 35 分钟
任务说明：模拟"误删数据"场景，使用二进制日志，基于时间戳恢复到误删之前状态。这是 DBA 必备技能！
跟着做
1
CREATE DATABASE test_recover; CREATE TABLE t1(id INT); INSERT INTO t1 VALUES(1),(2),(3);
2
SELECT NOW();（记录 T1）
3
INSERT INTO t1 VALUES(4),(5);
4
模拟误删：DELETE FROM t1 WHERE id>3;
5
mysqlbinlog --stop-datetime="T1" /var/lib/mysql/mysql-bin.000001 | mysql -u root -p
6
SELECT * FROM t1;（确认 id=4,5 恢复成功）

跟着做
1
mysqldump -u root -p --single-transaction test_recover > /tmp/backup.sql
2
DROP DATABASE test_recover;

2

任务 5-2：使用 Percona XtraBackup 备份和恢复

⭐ 核心任务 约 35 分钟
任务说明：使用 Percona XtraBackup 进行物理热备（不锁表），适合生产环境大型数据库备份，对比 mysqldump 效率差异。
跟着做
1
sudo yum install -y percona-xtrabackup-24
2
sudo xtrabackup --backup --target-dir=/backup/full --user=root --password=你的密码
3
sudo xtrabackup --prepare --target-dir=/backup/full
4
sudo xtrabackup --copy-back --target-dir=/backup/full
5
sudo chown -R mysql:mysql /var/lib/mysql && sudo systemctl start mysqld
3

任务 5-3：使用 mysqldump + crontab 自动备份

⭐ 核心任务 约 30 分钟
任务说明：编写备份脚本，配置 crontab，每天凌晨 2:00 自动备份，实现无人值守。
跟着做
1
编写 /root/backup_db.sh：mysqldump -u root -pMyPass2026! --all-databases > /backup/all_$(date +%Y%m%d).sql
2
chmod +x /root/backup_db.sh && bash /root/backup_db.sh
3
crontab -e，添加：0 2 * * * /root/backup_db.sh > /root/backup.log 2>&1
4
crontab -l（查看确认）
4

任务 5-4：迁移 MySQL 数据库

📝 实训任务 约 25 分钟
任务说明：将 Linux MySQL 数据库迁移到另一台服务器，完成跨平台数据迁移。
跟着做
1

2
scp /tmp/all.sql user@目标IP:/tmp/
3
mysql -u root -p < /tmp/all.sql && mysql -u root -p -e "SHOW DATABASES;"
